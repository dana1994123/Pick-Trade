We are planning to create an application that help people get rid of their unwanted clothing in trade of reward points that can be used to get clothing from other users. Our app will give the users the opportunities to add a list of clothing that they aim to get rid of and other users can send a request to take them. User has the ability to accept or decline the request based on their preference.

The presentation layer will compose of the following:

  1. 1- Home page: a page that will contains a list of clothing from everyone on the app.
  2. 2- Detailed clothing page: a page that will show more details about the item that the user chooses & the ability to send a request.
  3. 3- Items Listed: a page that will contains a list of the clothing that the user post on the app.
  4. 4- Request Page: a page that contains a list of requests received from other user to specific piece of clothing.

The business logic tier will have a couple of significant use cases:

  1. 1- Post a trade item
  2. 2- Accept/decline request
  3. 3- send request
  4. 4- Increment/decrement points
  5. 5- Browse trade items from all the users.

We are sure that our project will deserve a 100% since we have added all the required functionality and design the app in proper way.
